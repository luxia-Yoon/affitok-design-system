색각이상 접근성 (Color Blindness Accessibility)

전 세계 남성의 약 8%가 색각이상을 가진다. 색상만으로 정보를 전달하지 않는 것이 원칙이다.

색각이상 유형 및 유병률

Protanopia

~1%

Deuteranopia

~6%

녹색 인식 불가

가장 흔함. 빨강·녹색 혼동

Tritanopia

~0.01%

청색 인식 불가

파랑·노랑 혼동

~0.003%

3중 전달 (3-Layer Communication)

모든 상태 정보는 색상 + 아이콘 + 텍스트 3가지 채널로 동시에 전달한다.

상태별 3중 전달 구현 예시

Success, Error, Warning, Info 4가지 상태의 완전한 3중 전달 패턴이다.

크리에이터 등록 완료

시각 품질 셀프 체크 (Self-Audit Checklist)

컴포넌트 개발 완료 후 머지 전에 아래 12개 항목을 스스로 점검한다. 모두 통과해야 PR을 올린다.

구조 (Structure) — V1~V4

레이아웃과 시각적 위계 구조를 점검한다.

V1

시각적 위계가 명확하다

헤딩 > 소제목 > 본문 > 보조 텍스트 4단계가 구별되는지 확인

V2

간격 리듬이 일관하다

4px 배수 그리드 기준. 4, 8, 12, 16, 24, 32, 48px만 사용

V3

색상 대비 비율을 통과한다

Chrome DevTools Accessibility 패널 또는 contrast-ratio.com에서 확인

V4

3초 안에 목적을 파악할 수 있다

디자인을 처음 보는 팀원에게 보여주고 목적을 말할 수 있는지 확인

세부 (Detail) — V5~V8

데이터 표현과 일관성을 점검한다.

V5

데이터 그루핑이 논리적이다

관련 항목끼리 묶여 있고, 그룹 간 간격이 그룹 내 간격보다 넓은지 확인

V6

동일 컨텍스트 내 스타일이 일관하다

같은 역할을 하는 버튼·아이콘·색상이 동일한 스타일을 사용하는지 확인

V7

텍스트 위계가 올바르다

폰트 크기·굵기·색상이 중요도와 일치하는지 확인. 굵기 남용 금지

V8

숫자 데이터가 오른쪽 정렬이다

테이블 내 숫자 컬럼은 text-right. 천 단위 구분자(,) 포맷 확인

인터랙션 (Interaction) — V9~V12

상태 피드백과 접근성을 점검한다.

V9

호버·포커스 피드백이 명확하다

모든 클릭 가능 요소에 hover, focus-visible 스타일이 있는지 Tab으로 순회하며 확인

V10

빈 상태(Empty state)를 처리했다

데이터 없을 때 빈 화면이 아닌 안내 메시지+액션이 있는지 확인

V11

에러 상태를 처리했다

API 실패·입력 오류 시 색상+아이콘+텍스트 3중 에러 메시지가 있는지 확인

V12

키보드로 전체 조작이 가능하다

마우스 없이 Tab/Enter/Escape로 핵심 사용자 흐름을 끝까지 진행할 수 있는지 확인

채점 기준

PASS

12 / 12

모든 항목 통과. PR 제출 가능.

CONDITIONAL

10–11 / 12

미통과 항목을 이슈로 등록 후 PR 제출 가능.

FAIL

9 이하 / 12

재작업 후 다시 점검. PR 제출 불가.
